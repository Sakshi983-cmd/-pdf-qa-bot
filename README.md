📄 PDF Question Answering (PDF QA) Bot

Live App Link

A Streamlit-based web application that allows users to upload PDF documents and ask contextual questions. The bot intelligently reads PDFs, performs semantic search, and generates accurate answers.

🚀 Features

📥 Upload multiple PDF files at once (up to 200 MB each)

🧠 Answer questions based on PDF content

🔍 Semantic search using embeddings

⚡ Fast responses using Hugging Face Transformers

🌐 Deployed on Streamlit Cloud for instant access

🛠️ Technologies Used

Streamlit – Web interface

pdfplumber – PDF extraction

SentenceTransformers (all-MiniLM-L6-v2) – Semantic embeddings

Hugging Face Transformers (roberta-base-squad2) – Question-answering model

FAISS – Efficient similarity search

🔄 Implementation Flow
flowchart TD
    A[Upload PDF] --> B[Extract Text using pdfplumber]
    B --> C[Generate embeddings with SentenceTransformers]
    C --> D[Store embeddings in FAISS index]
    D --> E[User asks question]
    E --> F[Search relevant context using FAISS]
    F --> G[Answer generated by RoBERTa QA model]
    G --> H[Display answer in Streamlit]


Users upload PDFs

Bot extracts text from PDFs

Semantic embeddings are generated

Relevant sections are retrieved using FAISS

QA model generates answers

Streamlit displays results with confidence scores

📌 Usage

Upload one or more PDF files.

Enter a question in the input field.

The bot returns an answer with a confidence score and preview of the context.

📊 Output Example

Question: "What is the conclusion of the paper?"

Answer: "Top quality and accurate products which will raise the company's reputation and improve digital processing."

Confidence Score: 0.0318

Context Preview: Includes relevant PDF section for reference.
